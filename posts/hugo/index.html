Setup

create a new site

hugo new site blog
  • site의 내용을 구성할 blog 폴더가 생성됨

add a theme

  • https://themes.gohugo.io
cd blog
git init
git submodule add https://github.com/budparr/gohugo-theme-ananke.git themes/ananke
// themes/ananke 폴더의 remote origin이 https://github.com/budparr/gohugo-theme-ananke.git이 됨
  • ananke theme를 적용(submodule로 적용)
echo 'theme = "ananke"' >> config.toml
  • themes 폴더에 적용할 theme를 clone하지 않고 submodule로 적용하는 이유
    • submodule로 적용하면 업데이트된 theme의 내용을 쉽게 가져올 수 있다.

add some content

hugo new posts/my-first-post.md

  • blog/contents/posts 폴더에 my-first-post.md 파일이 성성됨

  • my-first-post.md 파일 내용

---
title: "my-first-post"
date: 2020-08-31T18:03:36+09:00
draft: true
---
  • draft: true : 이 문서가 draft 문서임을 나타냄 (나중에 release할 때는 이 부분을 제거하거나 draft: false로 변경한다)

start Hugo server

  • 문서를 작업 중에는 변경 내용을 확인하기 위해서 Hugo 서버를 실행서켜서 로컬에서 확인할 수 있다.
  • 이후에 Github Pages를 이용해서 서빙할 때는 Hugo 서버를 실행시킬 필요가 없다.
hugo server -D
  • -D : draft 문서도 볼 수 있게하는 옵션
  • http://localhost:1313/에서 페이지를 확인할 수 있다.
  • Hugo 서버를 다시 실행시키지 않아도 변경된 문서의 내용이 자동으로 반영된다.

build static pages

  • 작업한 문서를 웹 브라우저로 볼 수 있는 static page로 변환(build)해준다.
hugo -D
  • -D : draft 문서도 포함해서 build를 수행하는 옵션
  • build를 수행하면 defautl로 blog/public 폴더에 build된 내용이 생성된다.

Theme Config (for Ananke)

  • https://themes.gohugo.io/gohugo-theme-ananke/
  • blog/config.toml을 blog/theme/ananke/exampleSite/config.toml으로 대체
  • delete themeDir = "../.." line
title = "your tile..."
baseURL = "https://[github id].github.io/"
languageCode = "en-us"
theme = "ananke"

DefaultContentLanguage = "en"
disqusShortname = "[github id]"

[params]
  background_color_class = "bg-navy"
  featured_image = "/images/gohugo-default-sample-hero-image.jpg"
  recent_posts_number = 5
  commentEnable = true
  • site log (좌측 상단 로고)
    • 로고 이미지를 blog/static 폴더에 위치시켜야함
    • config.toml에 이미지 경로 설정
[params]
  site_logo = "img/logo.svg"

article edit with Ananke Theme

---
date: 2017-04-09T10:58:08-04:00
description: "The Grand Hall"
featured_image: "/images/Pope-Edouard-de-Beaumont-1844.jpg"
tags: ["scene"]
title: "Chapter I: The Grand Hall"
---
  • description : title 아래에 표시되는 설명문
  • featured_image : 해당 post를 대표하는 이미지 (없으면, featured_image: “")
  • tags: post된 글들 중에서 동일한 tag를 갖는 것들만 모아서 볼 수 있다.
    • 여러 개를 선언할 수 있다. (없으면, tags: [])
    • 동일한 tag를 가지고 있는 post가 있으면 글 오른쪽에 “Related” 박스가 보임
  • 글 본문에 이미지 넣기

Github 연동

  • github.com에 두 개의 repository 생성

    • blog
      • site repository (private repository)
      • Hugo의 컨텐츠와 소스 파일을 저장하는 repository (hugo new site blog로 새로운 사이트를 만들었다고 가정)
    • {github_id}.github.io
      • rendering repository (public repository)
      • web page로 렌더링할 컨텐츠를 가지는 repository

  • site repository를 dragon 폴더의 remote로 등록 & push

cd blog
git remote add origin https://github.com/{github_id}/blog.git
echo 'themes/' >> .gitignore    // theme는 submodule로 등록해 놓았기 때문
git add ...
git commit -m ...
git push origin master
  • blog/public 폴더의 remote origin을 https://github.com/{github_id}/blog.git으로 만든다.
git submodule add -b master https://github.com/{github_id}/{github_id}.github.io.git public
// -b : branch
  • https://{github_id}.github.io에서 내용을 확인할 수 있다.

Github Deploy

  • 새 파일 생성 & 컨텐츠 입력

    • hugo new post/test.md

  • build

    • hugo -D

  • site repository로 업로드

cd blog
git add .
git commit -m "..."
git push origin master
  • public folder upload ({github_id}.github.io repository로 업로드)
cd public
git add .
git commit -m "..."
git push origin master
  • deploy 과정을 스크립트로 작성해서 수행하면 편리함
    • https://gohugo.io/hosting-and-deployment/hosting-on-github/#put-it-into-a-script
